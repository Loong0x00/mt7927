## 3. MT6639 vs MT7925 固件架构差异

### Patch 固件对比

| 参数 | MT6639 | MT7925 (Windows) | MT7925 (Linux) | 差异 |
|------|--------|------------------|----------------|------|
| **总大小** | 293 KB | 209 KB | 194 KB | MT6639 比 MT7925 大 **40-50%** |
| **Region 0 Size** | 74944 B | 39744 B | 38912 B | MT6639 是 MT7925 的 **1.9x** |
| **Region 1 Size** | 224320 B | 174016 B | 158656 B | MT6639 是 MT7925 的 **1.4x** |
| **Region 1 Addr** | 0xe0020000 | 0xe0002800 | 0xe0002800 | **地址不同** |

### RAM 固件对比

| 参数 | MT6639 | MT7925 (Windows) | MT7925 (Linux) |
|------|--------|------------------|----------------|
| **总大小** | 1.6 MB | 1.1 MB | 1.2 MB |
| **差异** | MT6639 比 MT7925 大 **45%** | - | - |

### Header 结构（相同）

两种固件都使用相同的 header 结构：
- Platform: `ALPS`
- HW/SW ver: `0x8a108a10`
- Patch ver: `0xffffffff`
- Desc patch ver: `0x44332211`
- Desc subsys: `0x00000004`
- Number of regions: `2`

**关键差异总结**:
1. **Region 大小差异显著** — MT6639 包含更多代码/数据
2. **Region 1 加载地址不同** — MT6639: 0xe0020000 vs MT7925: 0xe0002800
3. **固件不可互换** — 使用错误的固件会导致硬件初始化失败

### Windows 驱动使用的固件推测

根据驱动包结构：
1. **MT7927 专用配置文件存在** (mtkwl7927.dat)
2. **MT6639 固件先于 MT7925** 在提取顺序中

**结论**: Windows 驱动对 MT7927 使用 **MT6639 固件** + **mtkwl7927.dat 配置**。

### 固件加密与初始化

两种固件的 Patch Region 都有：
- **Key idx**: `0x01000000` (encryption enabled)
- **Type**: `0x00030002` (可能包含加密类型标志)

驱动已正确设置：
- `DL_MODE_ENCRYPT | DL_MODE_RESET_SEC_IV` ✅

### 固件与 HOST RX 环的关系

**关键问题**: 固件是否检查 HOST RX ring 0 的存在？

- MT7925 驱动在 FWDL **之前**分配 HOST RX ring 0（512 entries）
- 我们的驱动在 Mode 53 **之后**才添加 ring 0（128 entries）
- **固件可能在启动时检测 ring 0 → 如果不存在，不配置 MCU_RX0**

**Mode 53 测试的意义**:
- ✅ 如果 Mode 53 成功 → 固件需要 HOST RX ring 0
- ❌ 如果 Mode 53 失败 → 问题在其他地方（DMA 路由、DMASHDL、MCU 命令格式）
